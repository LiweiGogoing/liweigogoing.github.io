Java+并发之volatile和CAS

发表于 2021-10-16 | 分类于 多线程

背景

讨论并发的时候基本会讨论两个问题：可见性和原子性，Java中线程安全除了使用Synchronize关键字还有Lock，而Lock的理论基础除了LockSupport还有volatile 和 CAS，本文主要介绍volatile和CAS的原理以及使用场景。

volatile

volatile关键字常见于单例的双重检查中，volatile关键字主要有如下作用：

  1. 保证多线程的可见性。
  2. 防止指令重排序。
可见性

随着CPU架构的不断进步，CPU为了提高程序运行效率增加多级缓存机制，读取数据的顺序可能如下：

CPU -> 寄存器 -> 多级缓存 -> 主存 -> 磁盘

volatile是编译器在编译的时候使用内存屏障确保可见性，这里没有细致研究，可以看下知乎上面的文章。

指令重排序

指令重排序也是为了提高程序整体的运行效率设计的，简单解释为：只要最终的运行结果不变，中间的指令可以任意变换顺序。

一般情况下在资源竞争没有那么激烈的情况下，使用CAS能提高性能，主要是减少了线程切换的代价，但是增加了CPU的消耗。整体总结来说，有如下特点：

  1. CAS不停地使用CPU compare and swap，通过消耗CPU而不是系统切换线程，在资源竞争不激烈的情况下能提高性能。
  2. CAS不是加锁机制，是一个原子的尝试操作，有成功也有失败，所以需要上层根据结果做处理。
建议

在java中不建议直接使用CAS，可以使用CAS封装好的工具类，如Lock和Atomic工具，这些工具里面均使用了CAS机制。
